ANALISIS DAN MITIGASI KERENTANAN SQL INJECTION PADA DVWA MENGGUNAKAN STANDAR OWASP A03:2021

Proyek ini berfokus pada analisis kerentanan SQL Injection pada aplikasi Damn Vulnerable Web Application (DVWA) sebagai studi kasus, dengan mengacu pada standar keamanan OWASP Top 10 A03:2021 (Injection). Analisis dilakukan dengan menguji berbagai teknik SQL Injection, seperti tautology-based injection, union-based injection, dan error-based injection, untuk memahami bagaimana input pengguna yang tidak tervalidasi dapat dimanfaatkan untuk mengakses atau memanipulasi data secara tidak sah. Hasil pengujian menunjukkan bahwa kerentanan SQL Injection terjadi akibat:

  1. 1. Tidak adanya validasi dan sanitasi input
  2. 2.Penggunaan query SQL dinamis
  3. 3.Tidak diterapkannya mekanisme keamanan database yang memadai

    4. Sebagai langkah mitigasi, diterapkan beberapa solusi keamanan, antara lain:
  4. 1. Penggunaan Prepared Statements (Parameterized Queries)
  5. 2. Validasi dan sanitasi input pengguna
  6. 3. Pembatasan hak akses database (Least Privilege)
  7. 4. Penanganan error yang aman untuk mencegah kebocoran informasi
  8. 5. praktik secure coding sesuai standar OWASP
    9. Proyek ini bertujuan meningkatkan pemahaman tentang risiko SQL Injection serta menunjukkan pentingnya penerapan standar keamanan aplikasi web untuk melindungi data dan sistem dari serangan injeksi..
